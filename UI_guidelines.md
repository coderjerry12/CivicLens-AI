# UI_GUIDELINES.md

# Design Philosophy

Build a premium AI SaaS application.

The interface should be:

* Modern
* Clean
* Minimal
* Interactive
* Professional
* Responsive
* Fast

The design should feel similar in quality and polish to the provided inspiration but must be an original implementation.

Reference Inspiration:

https://parkflow-ai.vercel.app

Do not copy layouts directly.

Use the inspiration only for understanding design language.

# Components

Every page should reuse:

* Buttons
* Cards
* Inputs
* Dialogs
* Drawers
* Badges
* Toasts
* Tables

Never create multiple styles for the same component.

---

# Animations

Use smooth transitions.

Hover effects

Scale animations

Fade animations

Slide transitions

Skeleton loading

Micro-interactions

Do not use excessive animations.

---

# Dashboard

Use:

* KPI Cards
* Charts
* Activity Timeline
* Quick Actions
* Interactive Tables

Spacing should be generous.

---

# Forms

Large inputs

Clear labels

Helpful placeholders

Inline validation

Accessible error messages

---

# Empty States

Every empty state should contain:

* Icon
* Title
* Description
* Primary Action

---

# Loading States

Every async operation must have:

* Skeleton loader
* Disabled button
* Spinner

---

# Error States

Display user-friendly messages.

Never expose technical errors.

---

# Mobile

Responsive from the beginning.

Support:

Desktop

Tablet

Mobile

---

# Accessibility

Keyboard navigation

Focus states

ARIA labels where applicable

Readable color contrast

---

# General Rule

Every new screen must feel like it belongs to the same application.

Consistency is more important than visual complexity.
